CSS에는 16진수 코드, RGB(), HSL(), LCH() 등 색상을 나타낼 수 있는 다양한 방법이 존재한다.

한번쯤 찾아보고 싶어서 정리해 봤다.

# 기존 색상 형식

## Named colors

HTML에는 140개의 이름이 지정된 색상이 있다. `red`, `green`과 같은 색상 이름부터  `dodgerblue` , `hotpink` , `tomato` 와 같은 특별한 키워드도 존재한다.

이 컬러 키워드는 다음과 같은 사이트들에서 확인할 수 있다.

https://www.w3schools.com/cssref/css_colors.php

https://arantius.github.io/web-color-wheel/

색상 이름 방식은 임시 색상이 필요할 때 유용하지만 실무에서는 잘 사용하지 않는다.

또한, 140개의 웹 색상이 HTML 4 스펙이나 X11 유닉스 윈도우 시스템 등 다양한 곳에서 가져왔기 때문에 gray가 darkgray보다 짙은 색을 띄는 것과 같이 일관성 문제가 있다.

## rgb()

rgb는 red, green, blue의 약어로, 가장 추상화되지 않은 형식이다. 다음과 같이 쓴다.

```html
<style>
  body {
    color: rgb(255 0 0);
  }
</style>

<p>
  Hello world!
</p>
```

컴퓨터나 스마트폰 화면은 수백만 개의 작은 빨강, 초록, 파랑 LED로 구성된 픽셀로 이루어져 있다. RGB 색상 형식은 이러한 LED의 밝기를 직접 조절할 수 있게 해준다.

빨강, 초록, 파랑 각 값은 ‘채널’이라고 불리며, 각 채널은 0부터 255까지의 값을 가진다. 이 채널들을 다양한 비율로 혼합하여 1,600만 가지 이상의 색상을 만들 수 있다.

RGB는 빛의 물리학에 기반하여 빨강, 초록, 파랑 빛을 혼합해서 모든 색상을 만들 수 있다. (세 색의 값을 모두 255로 설정하면 흰색이 되고 모두 0으로 설정하면 검은색이 됨)

RGB 형식은 투명도를 제어하기 위한 네 번째 값인 '알파 채널'을 추가로 지정할 수 있다.

```html
<style>
  .translucent-box {
    background: rgb(255 0 0 / 0.5);
  }
</style>

<section>
  <div class="translucent-box"></div>
</section>
```

알파 채널 값은 0(완전히 투명)부터 1(완전히 불투명)까지의 범위를 가진다.

> [!NOTE]
> 
> 기존 CSS에서는 rgba 함수를 사용하며 투명도를 나타냈었다.
> 
> ```html
> .translucent-box {
>   background: rgba(255, 0, 0, 0.5);
> }
> ```
> 
> 그러나 CSS Colors 레벨 4 명세에서 이러한 방식이 변경되었으며, 모든 색상 형식 전반에 걸쳐 사용되는 표준화된 표기법이 도입되었다. (rgba가 사용 중단된 것은 아니지만 새로운 형식을 사용하는 것이 권장된다.)
>
> 투명도를 지정하는 새로운 방법은 다음과 같다.
> 
> ```html
> .translucent-box {
>   background: rgb(255 0 0 / 0.5);
> }
> ```
> 여기서 슬래시(/)는 구분 기호일 뿐 나눗셈과는 관련이 없다.

## HEX 코드

웹에서 가장 흔하게 사용되는 색상 형식이다. 다음과 같이 쓴다.

```html
<style>
  body {
    color: #FF0000;
  }
</style>

<p>
  Hello world!
</p>
```

6자리 코드는 각 채널(빨강/녹색/파랑)에 해당하는 16진수 2자리 값 세 개로 구성된다.

따라서 이는 rgb()와 마찬가지로 각 R/G/B 채널에 대해 0에서 255 사이의 값을 지정하는 것과 같다.

알파 채널을 포함시키고 싶다면 뒷자리에 2자리 16진수 코드를 포함시켜 전달할 수 있다.

```html
<style>
  .translucent-box {
    background: #FFFF0080;
  }
</style>
```

위 예시에서는 알파 채널 값을 80으로 지정했는데 이는 십진수로 128에 해당하므로 50% 불투명해진다.

## hsl()

위의 두 가지 색상 형식이 모두 빨강/녹색/파랑 채널에 특정 값을 전달하는 아이디어를 서로 다른 방식으로 표현한 것이라면, HSL은 색조(Hue), 채도(Saturation), 밝기(Lightness)를 바탕으로 색상을 정의하는 다른 색상 체계이다. 문법은 다음과 같다.

```css
.box {
  background-color: hsl(4deg 8% 2%);
}
```

괄호 안의 값은 다음과 같다.

- 색조(Hue) : 사용하고자 하는 색상의 안료를 나타낸다. 값은 0에서 360도까지이며, 원형 척도이므로 0도와 360도가 같은 빨간색을 나타낸다.
- 채도(Saturation) : 색상의 안료 양을 나타낸다. 값은 0%에서 100%까지이며, 0%는 완전히 회색조이고 100%는 가장 생생한 색상이다.
- 밝기(Lightness) : 색상의 밝고 어두움을 나타낸다. 값은 0%에서 100%까지이며, 0%는 완전한 검은색, 100%는 순수한 흰색이다.

이는 rgb()나 HEX처럼 밝기 값을 직접 제어하는 대신, 인간이 일반적으로 색상을 생각하는 방식과 더 가깝게 추상화되어 직관적이다.

RGB와 마찬가지로 / 구분자를 사용하여 투명도를 지정할 수 있다.

```css
 .translucent-box {
    background: hsl(0deg 100% 50% / 0.5);
  }
```

### HSL vs. HSB

대부분의 그래픽 디자인 소프트웨어는 HSB라는 유사한 색상 형식을 사용한다.

HSL이 Hue-Saturation-Lightness라면, HSB는 Hue-Saturation-Brightness로 밝기 척도가 다르다.

HSL에서 밝기(Lightness)는 검정색과 흰색 사이의 척도로, 밝기 0%는 항상 검정색이고 100%는 항상 흰색이다.

반면 HSB에서 밝기(Brightness)는 마찬가지로 명암을 나타내는 척도이며 0%는 검정색이지만, 100%는 채도에 따라 색이 달라진다. 밝기 100%+채도 0%는 흰색이고, 밝기 100%+채도 100%는 밝고 선명한 색상을 나타낸다.

CSS에서는 hsl() 함수만 존재하기 때문에 hsb 모델을 사용할 수 없다.

# 최신 색상 형식

위의 색상 형식은 오래전부터 사용되어 왔다. (hsl()은 인터넷 익스플로러 9에서도 지원)

최근 CSS에 추가된 새로운 색상 형식은 다음과 같다.

## Display P3

지금까지 살펴본 모든 색상 형식(rgb, HEX 코드, hsl)은 모두 표준 RGB 색공간(=sRGB)에 속한다.

Display-P3은 표준 sRGB 색 공간을 확장하여 더 밝고 생생한 색상을 사용할 수 있게 해준다. 크기는 sRGB에 비해 50% 더 크며, 특히 녹색 영역이 크게 확장되었다.

<img width="1200" height="400" alt="image" src="https://github.com/user-attachments/assets/95da16f7-61f9-42c1-9b27-3109a7e34191" />


P3 색 공간을 사용하려면 `color()` 함수 포맷을 사용해야 한다. 구문은 다음과 같다.

```css
.box {
  background: color(display-p3 1 0 0);
}
```

`color()` 함수는 색 공간과 RGB 값 집합을 인수로 받는다. 0에서 255까지의 범위 대신 0에서 1까지의 십진수 값을 사용하며, `/ 알파값` 구문도 지원한다.

```css
color(display-p3 1 0.5 0);
color(display-p3 1 0.5 0 / .5);
```

## lch()

HSL 색상 체계는 원통형 색상 공간을 기반으로 하며, 모든 색상이 같은 범위의 채도(0~100%)를 가진다고 가정한다.

그러나 현실에서 디스플레이나 사람의 눈은 각 색조에 대해 서로 다른 최대 채도를 가지고 있다.

HSL은 색 공간을 변형하고 모든 색조가 동일한 최대 채도값을 갖도록 확장함으로써 이러한 복잡성을 숨기고, 결과적으로 사람이 보기에 정확하지 않은 L(밝기) 값을 나타낸다.

ex) HSL 색상 형식에서는 다음 두 색상이 같은 밝기 값(50%)을 나타낸다.

<img width="725" height="401" alt="image" src="https://github.com/user-attachments/assets/a45331a4-d7d2-49e1-8b14-85daba2054e5" />


그러나 인간의 감각으로는 노란색이 파란색보다 훨씬 밝게 느껴진다.

LCH는 이를 보완하여 인간의 지각에 균일하게 느껴지도록 설계된 색상 형식이다. 동일한 밝기를 가진 두 색상은 똑같이 밝게 느껴진다.

ex) LCH에서 55% 밝기를 가진 노란색과 파란색은 다음과 같다.

<img width="703" height="407" alt="image" src="https://github.com/user-attachments/assets/19fe62f4-ca21-4627-a02f-c0214a705941" />


LCH은 밝기(Lightness), 채도(Chroma), 색조(Hue)의 약자이다. 개념적으로는 HSL과 거의 유사하지만 두 가지 큰 차이점이 있다.

1. LCH 시스템은 인간의 지각을 우선시하므로 밝기가 같은 두 색상은 동일하게 밝게 느껴진다.
2. 특정 색 공간에 국한되지 않는다.

LCH는 크로마(채도) 값에 상한선을 두지 않으며, 따라서 특정 색 공간(sRGB 혹은 P3)에 구속되지 않는다.

따라서 디스플레이 기술이 계속 발전함에 따라 모니터가 점점 더 넓은 색 영역을 구현할 것으로 예상되며, LCH는 크로마 값을 높여 확장된 색상을 자동으로 참조할 수 있게 된다.

```css
.red-box {
  /* sRGB에서 빨간색으로 보임 */
  background: lch(50% 120 20);
  /*
    이 색상은 sRGB에서는 동일한 색이지만,
    향후 출시될 더 넓은 색상의 디스플레이에서는 훨씬 더 붉게 보일 것임
  */
  background: lch(50% 500 20);
}
```

## oklch()

LCH에는 파란 계열 색상(색상 값 270~330 사이)에서 밝기나 채도를 조정할 때 예기치 않게 색상이 변하는 버그가 있다.

<img width="1100" height="443" alt="image" src="https://github.com/user-attachments/assets/803a0785-5b55-484d-81cd-be0aa6497817" />


위는 LCH와 OKLCH 색공간에서 같은 색상(hue)으로 고정한 단면을 비교한 것이다. LCH 단면은 한쪽은 파란색, 다른 쪽은 보라색으로 색상이 변한다.

OKLCH는 이러한 색조 이동 버그를 해결하기 위해 만들어진 LCH의 보완된 버전이며, 색역 보정이 개선되는 등 새로운 기능들도 함께 갖추고 있다.

- 색역 보정 : 모니터가 해당 색상을 직접 지원하지 않더라도 브라우저가 가장 가까운 색상으로 자동 보정해주는 것

OKLCH의 문법은 `oklch(L C H)` 또는 `oklch(L C H / a)` 형태로 표현된다. 각 요소에 대한 설명은 다음과 같다.

- 밝기(Lightness) : 사람이 인식하는 밝기이다. 값의 범위는 0(검정)부터 1(흰색)까지이며, 0% ~ 100%의 백분율로도 입력할 수 있다. 그러나 % 포맷은 calc와 같은 계산식에서 제대로 작동하지 않는다.
- 채도(Chroma) : 0(회색)에서 시작해 이론적으로는 무한대까지 갈 수 있으나 실제로는 디스플레이의 색영역에 따라 제한이 있다. (P3이 sRGB보다 더 높은 채도값 사용) 각 색조마다 최대 채도값이 다르며, 현재 색역(P3/sRGB)에서는 max값이 0.37이다.
- 색조(Hue) : 값의 범위는 0부터 360까지이며 0도와 360도는 같은 색상을 나타낸다. 60deg처럼 단위를 명시할 수도 있고, 그냥 60처럼 숫자만 써도 된다.
- 불투명도(alpha) : 0-1 또는 0~100%로 나타낸다.

```css
oklch(40.1% 0.123 21.57)
oklch(59.69% 0.156 49.77)
oklch(59.69% 0.156 49.77 / .5)
```

# 권장하는 색상 형식

RGB나 16진수 코드는 다음과 같은 단점이 있다.

- 낮은 직관성 : `#0F52B7`, `#F32AB9` 같은 표기는 16진수 표기법에 익숙하다고 해도 해독하기 어렵다.
- 값 조정의 어려움 : 색상을 조금 더 어둡게 하고 싶거나 선명하게 하고 싶을 때 값 수정만으로 색상을 조정하기 어렵다.

반면 HSL이나 OKLCH 같은 색상 표기법은 비교적 직관적이며, 기본값을 바탕으로 색상 변환을 할 때도 유용하다.

```css
html {
  --dark-red: hsl(
    var(--red-hue)
    var(--red-sat)
    calc(var(--red-lit) - 20%)
  );
}
```

HSL과 OKLCH의 장단점은 다음과 같다.

- HSL : 등장한 지 오래되어 브라우저 지원율이 높다(97%). 그러나 앞서 말했듯 명도가 인간의 직관적인 기준과 달라 색상 변화가 불완전하고, P3 색역을 지원하지 않는다.
- OKLCH : 넓은 색역과 예측 가능한 색상 변화를 지원한다. 단점은 비교적 최신 명세이기 때문에 HSL에 비해 브라우저 지원율이 낮다. (브라우저 지원율 92%)

신규 프로젝트라면 OKLCH가 좋은 선택으로 보인다.
